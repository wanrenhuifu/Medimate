# 10 - 数据策略

## 10.1 数据来源总览

| 数据类型 | 来源 | 更新方式 | 存储 | 可靠性 |
|---------|------|---------|------|--------|
| 药物信息 | 药品说明书（NMPA/丁香园/药智网） | RAG 向量检索 | Supabase pgvector | 高（说明书原文） |
| 药物相互作用 | RAG 语义检索 + 药品说明书 | 实时检索 | Supabase pgvector | 高（说明书原文） |
| 不良反应报告 | OpenFDA FAERS API | 实时查询 | API JSON | 高（FDA 官方数据） |
| 用户用药清单 | 用户自行录入 | 实时 | Supabase PostgreSQL | — |

## 10.2 RAG 知识库设计

MediMate 采用 **RAG（检索增强生成）** 架构：

```
药品说明书（公开数据源）
    ↓ 文本切片（按说明书标题层级结构化切片，保留章节完整性）
    ↓ BGE 中文嵌入（BAAI/bge-m3，1024维）
    ↓ 存入 Supabase pgvector
    ↓ 用户查询时语义检索 Top-K 相似段落
    ↓ 喂给 LLM 生成准确回答
```

### 覆盖范围

收集 **200+ 种常用药** 的说明书文本：

| 类别 | 预计数量 | 示例 |
|------|---------|------|
| 解热镇痛药 | 10-15 | 布洛芬、对乙酰氨基酚、阿司匹林 |
| 抗生素 | 15-20 | 阿莫西林、头孢氨苄、阿奇霉素 |
| 心血管药物 | 25-30 | 氨氯地平、美托洛尔、华法林 |
| 消化系统药物 | 10-15 | 奥美拉唑、泮托拉唑 |
| 呼吸系统药物 | 10-15 | 氯雷他定、沙丁胺醇 |
| 神经系统药物 | 10-15 | 舍曲林、加巴喷丁 |
| 降糖药 | 8-10 | 二甲双胍、甘精胰岛素 |
| 其他常用药 | 15-20 | 维生素 D3、叶酸、碳酸钙 |

### 切片数据结构

| 字段 | 说明 |
|------|------|
| name_cn | 中文通用名 |
| name_en | 英文通用名 |
| aliases | 别名/商品名数组 |
| chunk_text | 说明书原文片段（200-500字） |
| embedding | BGE-M3 1024维向量 |
| source | 来源文件名 |

## 10.3 嵌入模型

- **模型**: BAAI/bge-m3
- **维度**: 1024
- **语言**: 多语言优化（中文优先）
- **部署**: API 模式（兼容 OpenAI 嵌入接口）

## 10.4 OpenFDA API 数据策略

不良反应数据通过 OpenFDA FAERS API 实时查询。

API 调用方式：
```
GET https://api.fda.gov/drug/event.json
  ?search=patient.drug.medicinalproduct:%22ASPIRIN%22
  &count=patient.reaction.reactionmeddrapt.exact
```

## 10.5 数据更新流程

```
新增药品说明书 → 放入 data/package_inserts/[药物名].txt
    ↓
运行 python -m rag.pipeline --input_dir data/package_inserts
    ↓
自动切片 → 嵌入 → 全量导入 pgvector（幂等，可重复执行）
```
