# 11 - 成功指标

## 11.1 产品指标

| 指标类别 | 指标名称 | 定义 | 目标 |
|---------|---------|------|---------|
| **功能覆盖** | 药物识别率 | 用户输入的药名能被正确识别的比例 | ≥85% |
| **功能覆盖** | 工具调用准确率 | LLM 正确判断调用哪个工具的比例 | ≥85% |
| **数据能力** | FDA API 调用成功率 | API 正常返回数据的比例 | ≥95% |
| **数据能力** | 交互检查覆盖率 | 常见药物组合被交互数据库覆盖的比例 | ≥70% |
| **安全** | 紧急症状拦截率 | 紧急关键词被成功拦截的比例 | 100% |
| **安全** | 免责声明展示率 | 每次建议回复附带免责提示的比例 | 100% |

## 11.2 用户体验指标

| 指标 | 衡量方式 | 目标 |
|------|---------|------|
| 首次交互完成率 | 用户是否完成至少一次完整查询 | ≥70% |
| 单次会话轮次 | 平均每次会话的对话轮数 | ≥3 轮 |
| 功能使用分布 | 三大核心功能的使用占比 | 均有覆盖 |
| 对话完成率 | 用户提问后收到有效回复的比例 | ≥90% |
| 响应时间 | 从输入到 Agent 回复的时间 | 本地 <500ms / API <3s |

## 11.3 演示效果指标（面试场景）

| 演示项 | 预期效果 | 面试官关注点 |
|--------|---------|-------------|
| 冷启动体验 | 打开即用，5 秒内进入对话 | "部署方便吗？" |
| 核心流程演示 | 3 分钟内演示完查询 + 交互 + 副作用三个核心流程 | "能解决什么问题？" |
| WOW Moment | FDA 数据可视化让面试官眼前一亮 | "和 ChatGPT 有什么区别？" |
| 产品思维体现 | 安全兜底 + 追问确认 + 数据透明 | "你怎么思考 AI 的边界？" |
| 边界情况处理 | 输入未收录的药物，优雅降级 | "遇到问题怎么办？" |
| 紧急拦截演示 | 输入"胸痛"触发就医提示 | "安全性怎么保障？" |

## 11.4 评估方法

### 测试用例覆盖

| 测试类别 | 用例数 | 示例 |
|---------|--------|------|
| 药物查询-正常 | 20 | "布洛芬是什么" |
| 药物查询-模糊 | 10 | "芬必得"、"美林" |
| 药物查询-未收录 | 5 | "某冷门药物" |
| 交互检查-高危 | 10 | "华法林和阿司匹林" |
| 交互检查-安全 | 5 | "维生素C和钙片" |
| 副作用查询 | 10 | "阿司匹林有什么副作用" |
| 紧急症状 | 10 | "吃药后胸痛" |
| 一般对话 | 5 | "你好"、"谢谢" |

### 评分标准

| 等级 | 定义 | 标准 |
|------|------|------|
| ✅ Pass | 完全符合预期 | 意图正确、回复完整、格式正确 |
| ⚠️ Partial | 基本可用 | 意图正确但回复不够完整 |
| ❌ Fail | 不符合预期 | 意图错误或回复有误 |
