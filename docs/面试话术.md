# 面试话术 — MediMate 智能用药安全助手

> 面向 AI PM / 全栈工程师 面试场景的完整话术参考。
> 建议面试前通读 2-3 遍，重点记住数据、差异化对比、架构亮点。

---

## 一、30 秒电梯陈述

> 我做了一个 AI 用药安全助手叫 MediMate。中国每年有 210 万份药品不良反应报告，其中约 15% 源于多药相互作用——患者在家吃药，没有人帮他们审方。
>
> MediMate 是一个基于 LLM Function Calling + RAG 双引擎的 AI Agent，用户用自然语言描述用药情况，Agent 自主调用工具查询药品说明书、检查药物冲突、拉取 FDA 真实不良反应数据。核心差异在于：不是让 LLM 凭记忆回答，而是让它调用真实数据库——从架构层面抑制幻觉。
>
> 全栈独立完成，Vue 3 + FastAPI + DeepSeek + Supabase pgvector，4 个 Function Calling 工具，SSE 流式输出，双层安全机制。

---

## 二、高频问题与建议回答

### Q1：为什么做这个项目？

> 我注意到几个数据：中国年药品不良反应报告 210 万份，约 15% 源于多药相互作用；老年患者多重用药（≥5 种）比例超过 40%。
>
> 现有的工具要么面向医生（丁香园/用药助手），要么只有搜索功能（百度）。C 端患者缺乏一个"翻译层"——把专业药学知识翻译成他们能理解的信息。
>
> 我想用 AI Agent 的方式来解决——通过 LLM Function Calling 让 Agent 自主调用工具查询真实数据库，而不是凭记忆回答。同时设计了双层安全机制和四级免责体系，体现医疗 AI 的边界意识。

**关键数据**（面试官容易记住）：
- 210 万份/年 药品不良反应报告
- 15% 源于药物相互作用
- 40%+ 老年患者多重用药

---

### Q2：和直接问 ChatGPT 有什么区别？

> 三点核心差异：
>
> **1. 可信度**：ChatGPT 可能幻觉出不存在的药物信息。MediMate 的 LLM 通过 Function Calling 调用 RAG 语义检索和 FDA API，只负责理解意图和组织语言，不凭记忆补充药物知识。检索结果可溯源至药品说明书原文。
>
> **2. 实时数据**：`query_side_effects` 工具实时调用 FDA FAERS 数据库（1968 年至今累计），展示的是真实世界报告数据，而非模型训练时的静态记忆。竞品展示"常见/偶见/罕见"，我展示"报告次数排序 + 条形图"。
>
> **3. 安全机制**：双层安全拦截——前置正则硬兜底（零容忍高危词零漏报）+ LLM 语义判断（理解"吃药后胸痛"vs"说明书上写可能胸痛"的区别）。外加四级免责声明体系。通用大模型没有这种领域特化设计。

**对比表**（可直接画在白板上）：

| 维度 | ChatGPT | MediMate |
|------|---------|----------|
| 知识来源 | 训练记忆（静态/可能幻觉） | RAG 检索 + FDA 实时 API |
| 推理链路 | 单轮文本生成 | 多轮 Function Calling 循环 |
| 安全机制 | 通用 RLHF 对齐 | 双层拦截 + 四级免责 |
| 可观测性 | 黑盒 | 每次工具调用可追踪 |

### Q3：你对 Agent 的理解是什么？

> Agent 的核心是"感知 → 决策 → 行动"的闭环，关键在于 Tool Use。
>
> 普通 Chatbot 只能生成文本。Agent 能根据用户意图主动调用外部工具——查数据库、调 API，然后基于工具返回的真实数据组织回复。
>
> 在 MediMate 中，我定义了 4 个工具，通过 DeepSeek 的 Function Calling 让 LLM 自主决定：什么时候调哪个工具、传什么参数。比如用户说"我新开了华法林"，LLM 不仅调用了 `manage_medication_list(add)` 把药加入清单，还主动调用 `check_interaction` 检查华法林与已有药物的冲突。这个"主动检查"的行为不是代码硬编码的，是 LLM 根据 System Prompt 指令自主决策的。
>
> Agent Core 只有 50 行代码——只做 Loop 控制，所有智能决策全量委托 LLM。这是"瘦 Agent 核心 + 胖 Prompt"的现代 Agent 设计范式。

**关键概念**（如果面试官追问）：
- 循环上限 5 轮，防止无限工具调用
- 同轮多个 tool_calls 并行执行（`asyncio.gather`）
- 上下文管理：20 轮滑动窗口 + System Prompt 常驻

### Q4：怎么保证医疗安全？

> 我从两个层面设计安全机制：

> **第一层——架构层**：LLM 只摘要工具返回的数据，不凭记忆补充。System Prompt 显式禁止编造药物信息、提供处方建议、调整用药方案。这从根上减少幻觉。

> **第二层——安全拦截层（双重保险）**：
> - Layer 1 — 正则硬兜底：对零容忍高危词（自杀、胸痛、呼吸困难、大出血等）做前置正则拦截，命中直接覆写为就医引导模板。确定性代码，零漏报。
> - Layer 2 — LLM 语义判断：System Prompt 中以最高优先级声明安全规则，LLM 基于上下文判断紧急程度。能区分"吃药后胸痛"（紧急）和"说明书上写了可能胸痛"（信息查询）——这是纯正则做不到的。
>
> 此外还有四级免责声明体系：全局底部常驻 → 首次模态框确认 → 每条回复末尾追加 → FDA 数据标注"报告数≠发生概率"。

**如果面试官追问"为什么不用纯正则"**：
> LLM 能理解语义和上下文——"吃了一瓶药想自杀" vs "研究自杀基因的论文"、"我胸口有点闷" vs "说明书上写的胸闷"——LLM 能区分紧急程度，正则做不到。反过来，正则可以 100% 拦截"我想死"这种直白表达。所以两者互补：正则做零漏报，LLM 做低误报。

---

### Q5：这个项目你最自豪的设计是什么？

> 最自豪的是 **Function Calling 驱动的多工具编排**。
>
> 市面上很多"AI Agent"其实只是套壳 Chatbot。MediMate 是真正的 Agent——LLM 不只生成文本，它自主决策调哪个工具。
>
> 举一个具体例子：用户说"我新开了华法林，帮我加到用药清单"。LLM 第一轮并行调用了三个工具：① `manage_medication_list(add, 华法林)` ② `check_interaction([华法林, 阿司匹林, 奥美拉唑])` ③ `query_side_effects(华法林)`。第二轮，当它发现华法林×阿司匹林是 🔴 严重交互后，主动追加追问用户是否知情。
>
> 这个"跨轮次多工具编排 + 主动风险感知"的行为完全由 LLM 自主完成，代码里没有任何编排逻辑。迭代一个新行为只需改 Prompt，不需要改代码。
>
> 另外，FDA 不良反应数据的条形图可视化也是 WOW Moment——用户第一次看到真实世界的副作用报告数据，从焦虑变成了理性。

### Q6：如果给你更多资源，你会怎么迭代？

> 三步走：

> **短期（1 个月）**：做成微信小程序，扩大用户触达。加入语音输入，适配老年用户群体。目前老年用户是最需要用药安全的群体，但打字门槛高。

> **中期（3 个月）**：接入中国 NMPA 药品数据，更贴合国内用药场景。增加服药提醒功能，从"被动查询"变成"主动守护"。

> **长期**：探索 B 端变现——为连锁药房提供 SaaS 版用药助手。药房药师每天要回答大量重复的用药咨询，AI Agent 可以处理 70% 的常见问题，大幅提效。另一个方向是健康险公司的用药风险评估。

## 四、架构亮点速记

面试中如果被问到技术细节，以下亮点可直接展开：

| 亮点 | 一句话 | 展开方向 |
|------|--------|---------|
| **双引擎分流** | RAG 管知识，FC 管实时 | LLM 根据 System Prompt 自主路由，无硬编码分支 |
| **结构化切片** | 按说明书章节切，非字符数 | 用户问"孕妇能吃吗"→ 直接命中"禁忌"块，不会被"用法用量"干扰 |

| **上下文管理** | 20 轮滑动窗口 + 常驻 System Prompt | LLM 天然指代消解，无需额外 NLP |
| **胖 Prompt 瘦 Agent** | Agent Core 只有循环逻辑 | 改行为 = 改 Prompt，迭代成本趋近于零 |
| **BGE-M3 嵌入** | 1024 维多语言向量 | API 模式部署，兼容 OpenAI 嵌入接口 |

## 七、常见追问与应对

### "你怎么评估 Agent 的质量？"
> 我定义了 8 类 75 条测试用例（正常查询、模糊匹配、高危 DDI、紧急症状等），三级评分（Pass/Partial/Fail）。核心指标：意图识别准确率 ≥85%、工具调用成功率 ≥95%、幻觉率 <5%、紧急拦截率 100%。

### "为什么选 DeepSeek 而不是 OpenAI？"
> 成本是 GPT-4o 的 1/50，中文效果优秀，Function Calling 完整兼容 OpenAI 接口格式。架构上不绑定模型——换个 API Key 就能切换。

### "为什么选 Vue 而不是 React？"
> 国内就业市场 Vue 占有率更高，简历关键词匹配度好。单文件组件开发效率高，官方全家桶（Router + Pinia + Vite）生态完整。

### "为什么选 FastAPI 而不是 Express/Spring？"
> Python 是 AI/ML 第一语言，面试加分。原生 async/await 适合 SSE 流式输出。Pydantic 自动校验，自带 Swagger 文档。

### "如果做成真正的商业产品，最大的挑战是什么？"
> 合规和数据覆盖。医疗 AI 的监管边界需要明确——我们的定位是"信息查询工具"而非"医疗设备"。数据方面需要对接 NMPA 中国药品数据库，目前用的是 FDA 美国数据。另外药品说明书授权的法律问题也需要解决。
