# 14 - 附录

## A. 技术栈选型

| 层次 | 技术选型 | 选型理由 |
|------|---------|---------|
| 前端框架 | Vue 3 + Vite | 国内主流，生态成熟，简历关键词匹配度高 |
| 后端框架 | Python FastAPI | AI 方向标配语言，天然支持异步和 SSE |
| LLM | DeepSeek API (Function Calling) | 中文效果好、成本极低（~¥1/百万 Token）、兼容 OpenAI 接口 |
| 数据库 | Supabase PostgreSQL + pgvector | 云端托管，支持向量检索，便于扩展 |
| UI 风格 | 深色主题 + 玻璃拟态 | 现代感，医疗科技感 |
| 字体 | Google Fonts (Inter) | 清晰易读 |
| 外部 API | OpenFDA REST API | 免费、无需注册、数据权威 |
| 状态管理 | Pinia | Vue 3 官方推荐 |

### 为什么选 Vue 而不是 React？

| 考虑 | 分析 |
|------|------|
| **国内就业市场** | Vue 在国内占有率更高，简历关键词匹配度好 |
| **上手速度** | 单文件组件更直观，开发效率高 |
| **生态完整** | Vue Router + Pinia + Vite 官方全家桶 |

### 为什么选 FastAPI 而不是 Express？

| 考虑 | 分析 |
|------|------|
| **AI 生态** | Python 是 AI/ML 第一语言，面试加分 |
| **异步支持** | 原生 async/await，适合 SSE 流式输出 |
| **类型安全** | Pydantic 自动校验请求/响应 |
| **文档自动生成** | 自带 Swagger UI，方便调试 |

### 为什么选 DeepSeek 而不是 OpenAI？

| 考虑 | 分析 |
|------|------|
| **成本** | 约 ¥1/百万 Token，是 GPT-4o 的 1/50 |
| **中文效果** | 中文理解和生成质量优秀 |
| **Function Calling** | 完整支持，兼容 OpenAI 接口格式 |
| **可替换性** | 接口兼容 OpenAI SDK，随时可切换到其他模型 |

## B. 面试话术建议

> 📋 **完整版面试话术已独立整理为** → [`面试话术.md`](面试话术.md)
> 包含：30 秒电梯陈述、6 问完整回答、产品关键数字、架构亮点速记、演示流程、常见追问应对。
> 以下为精简版关键词提纲，面试前建议通读完整版。

### Q1 为什么做
- 210 万份/年药品不良反应报告，15% 源于多药相互作用
- 现有工具面向医生，C 端缺乏"翻译层"
- 用 AI Agent 实现对话式审方

### Q2 和 ChatGPT 区别
- **知识来源**：RAG 检索 + FDA 实时 API vs 训练记忆
- **推理链路**：多轮 Function Calling 循环 vs 单轮文本生成
- **安全机制**：双层拦截 + 四级免责 vs 通用 RLHF 对齐

### Q3 对 Agent 的理解
- 核心是"感知 → 决策 → 行动"闭环，关键在于 Tool Use
- Agent Core 仅 50 行——瘦 Agent + 胖 Prompt 设计范式
- LLM 自主决定何时调哪个工具、传什么参数

### Q4 怎么保证安全
- Layer 1 正则硬兜底（零漏报）+ Layer 2 LLM 语义判断（低误报）
- 四级免责声明体系（全局 → 首次 → 回复级 → 数据级）
- LLM 不凭记忆补充药物知识，从架构层抑制幻觉

### Q5 最自豪的设计
- 跨轮次多工具编排——LLM 主动检查新加药物与已有清单的冲突
- FDA 真实世界数据可视化——从"常见/偶见/罕见"变为"报告次数排序"

### Q6 迭代方向
- 短期：微信小程序 + 语音输入
- 中期：NMPA 中国数据 + 服药提醒
- 长期：B 端 SaaS（连锁药房 + 保险增值）
